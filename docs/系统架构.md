## 1. 架构总览

CRA AI Assistant 采用 Electron 双进程架构，遵循最小权限原则，主进程负责业务逻辑和系统调用，渲染进程仅负责 UI 展示，通过预加载脚本的安全桥接层通信。

## 3. AI 网关架构

### 3.1 网关组件图

```
                    invokeGateway(request)
                           │
                           ▼
                    ┌──────────────┐
                    │ SmartRouter  │
                    │  (单例模式)   │
                    └──────┬───────┘
                           │
                    ┌──────┴───────┐
                    │  route()     │
                    │  根据 config │
                    │  选择适配器  │
                    └──┬───────┬───┘
                       │       │
            ┌──────────┘       └──────────┐
            ▼                              ▼
    ┌───────────────┐              ┌───────────────┐
    │ ZhipuAdapter  │              │ OpenAIAdapter │
    │               │              │               │
    │ chat()        │              │ chat()        │
    │ chatWithImage │              │ chatWithImage │
    └───────┬───────┘              └───────┬───────┘
            │                              │
            ▼                              ▼
    ┌───────────────┐              ┌───────────────┐
    │ Zhipu API     │              │ OpenAI API    │
    │ (bigmodel.cn) │              │ (兼容端点)     │
    └───────────────┘              └───────────────┘

路由规则:
  provider=zhipu  + text  → ZhipuAdapter.chat()        → textModel (glm-5.1)
  provider=zhipu  + image → ZhipuAdapter.chatWithImage() → visionModel (glm-4.6v-flash)
  provider=openai + text  → OpenAIAdapter.chat()        → openaiModel (gpt-4o-mini)
  provider=openai + image → OpenAIAdapter.chatWithImage() → openaiModel (gpt-4o)
```

### 3.2 请求/响应数据流

```
Agent (调用方)
  │
  │  GatewayRequest {
  │    prompt: string,
  │    systemPrompt?: string,
  │    contentType: 'text' | 'image',
  │    imageBase64?: string,      ← 图片时必填
  │    imageMimeType?: string,    ← 图片时必填
  │    temperature?: number,      ← 默认 0.1
  │    maxTokens?: number         ← 默认 4096
  │  }
  │
  ▼
SmartRouter.route(request)
  │  → { adapter, model }
  ▼
Adapter.invoke(request)
  │  → adapter.chat(prompt, systemPrompt, options)
  │  或 adapter.chatWithImage(prompt, image, mime, systemPrompt, options)
  │
  │  HTTP POST (axios)
  │  Authorization: Bearer <apiKey>
  │  Content-Type: application/json
  │  Body: { model, messages, temperature, max_tokens }
  │
  ▼
API Server
  │
  │  ZhipuResponse {
  │    choices: [{ message: { content, reasoning_content }, finish_reason }],
  │    usage: { prompt_tokens, completion_tokens, total_tokens },
  │    model: string
  │  }
  │
  ▼
GatewayResponse {
    content: string,
    model: string,
    usage: { promptTokens, completionTokens, totalTokens }
  }
```
