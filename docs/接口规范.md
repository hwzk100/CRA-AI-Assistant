# CRA AI Assistant 接口规范文档

> 版本：5.0.1 | 最后更新：2026-04-20

---

## 1. 概述

本文档定义 CRA AI Assistant 中所有跨进程通信接口（IPC）、内部模块接口和外部 API 调用规范。

### 1.1 通信层分类

| 层级 | 通信方式 | 定义位置 |
|------|---------|---------|
| 渲染进程 ↔ 主进程 | Electron IPC (`ipcRenderer.invoke` / `ipcMain.handle`) | `shared/constants/ipc-channels.ts` |
| 主进程 → 渲染进程 | Electron Event (`webContents.send`) | `shared/constants/ipc-channels.ts` |
| Agent → AI 模型 | HTTP REST API (axios) | `shared/types/gateway.ts` |
| 主进程内部 | TypeScript 函数调用 | 各 Agent 模块 |

## 2. IPC 接口规范

### 2.1 getSettings

获取当前应用配置。

| 项目 | 值 |
|------|-----|
| 通道 | `get-settings` |
| 方向 | 渲染进程 → 主进程 |
| 请求参数 | 无 |
| 响应类型 | `AppConfig` |

**响应结构 `AppConfig`**：

```typescript
{
  apiEndpoint: string;       // AI API 端点 URL
  apiKey: string;            // API 密钥（来自 .env 或加密存储）
  textModel: string;         // 文本模型名称
  visionModel: string;       // 视觉模型名称
  provider: 'zhipu' | 'openai';  // 当前服务商
  openaiApiEndpoint?: string;    // OpenAI 端点（可选）
  openaiApiKey?: string;         // OpenAI 密钥（可选）
  openaiModel?: string;          // OpenAI 模型（可选）
}
```

### 2.2 saveSettings

保存应用配置。

| 项目 | 值 |
|------|-----|
| 通道 | `save-settings` |
| 方向 | 渲染进程 → 主进程 |
| 请求参数 | `Partial<AppConfig>` |
| 响应类型 | `AppConfig` |

**行为说明**：
- API 密钥通过 `electron.safeStorage` 加密后单独存储

- 保存后自动调用 `resetRouter()` 使新配置生效

| 项目 | 值 |
|------|-----|
| 通道 | `test-connection` |
| 方向 | 渲染进程 → 主进程 |
| 请求参数 | 无 |
| 响应类型 | `ConnectionTestResult` |

**响应结构 `ConnectionTestResult`**：

```typescript
{
  success: boolean;       // 是否成功
  message: string;        // 结果消息（成功："连接成功: xxx"，失败："连接失败: xxx"）
  model?: string;         // 响应模型名称
```

**行为说明**：

- 发送提示词 `"请回复"连接成功"四个字。"` 到 AI 模型
- `maxTokens: 20`，`temperature: 0`

### 2.4 uploadProtocolFile

上传并验证方案文件。

| 项目 | 值 |
|------|-----|
| 通道 | `upload-protocol-file` |
| 方向 | 渲染进程 → 主进程 |
| 请求参数 | `filePath: string` |
| 响应类型 | `FileUploadResult` |

**请求参数**：

```typescript
filePath: string  // 文件绝对路径
```

**响应结构 `FileUploadResult`**：

```typescript
{
  success: boolean;                          // 是否成功
  filePath: string;                          // 文件路径
  fileName: string;                          // 文件名
  fileType: 'text-pdf' | 'scanned-pdf' | 'image';  // 检测到的文件类型
  message?: string;                          // 错误消息（失败时）
}
```

**行为说明**：
- 验证文件路径（防路径遍历、存在性、MIME 类型、大小）
- 检测文件类型（text-pdf / scanned-pdf / image）
- 将文件信息存储到内存变量 `protocolFileInfo`
- 通过 `PROCESSING_PROGRESS` 推送进度

**进度事件**：

| 阶段 | 进度 | 消息 |
|------|------|------|
| uploading | 0 | 正在验证方案文件... |
| complete | 100 | 文件上传完成 |
| error | 0 | 上传失败: {错误信息} |

### 2.6 extractCriteria

从已上传的方案文件中 AI 提取入排标准和访视计划。

| 项目 | 值 |
|------|-----|
| 通道 | `extract-criteria` |
| 方向 | 渲染进程 → 主进程 |
| 请求参数 | 无（使用内存中的 `protocolFileInfo`） |
| 响应类型 | `ProtocolData` |

**前置条件**：必须先调用 `uploadProtocolFile` 成功。

**响应结构 `ProtocolData`**：

```typescript
{
  criteria: Criterion[];          // 入排标准列表
  visitSchedules: VisitSchedule[]; // 访视计划列表
  protocolTitle?: string;         // 方案标题（可选）
  protocolId?: string;            // 方案编号（可选）
  rawText?: string;               // 原始文本（可选）
}
```

**`Criterion` 结构**：

```typescript
{
  id: string;                          // "criterion_{timestamp}_{random}"
  category: 'inclusion' | 'exclusion';  // 标准 category
  index: number;                       // 原文序号
  content: string;                     // 标准内容
  source?: string;                     // 来源（可选）
}
```

**`VisitSchedule` 结构**：

```typescript
{
  id: string;                    // "visit_{timestamp}_{random}"
  visitName: string;             // 访视名称
  visitWindow: string;           // 访视窗口
  procedures: string[];          // 检查项目列表
  timing: string;                // 时间安排
}
```

**处理策略**：

| 文件类型 | 策略 |
|----------|------|
| text-pdf | 解析文本 → 动态分块(`getChunkTokens()`) → 关键词预筛选 → 每块提取标准 + 访视 → 合并去重 |
| scanned-pdf | PDF 转图片 → 按 `getMaxImagesPerCall()` 批量分组 → 每组提取标准 → 合并去重（不提取访视） |
| image | 单次提取标准（不提取访视） |

**关键词预筛选**（仅 text-pdf）：
- 关键词：纳入标准、排除标准、入选标准、入排标准、访视、随访、inclusion、exclusion 等
- 匹配到的分块及其相邻 ±1 块会被保留用于 AI 处理
- 无关键词命中时退回处理所有分块

**去重规则**：
- 标准：按内容前 100 字符归一化比较
- 访视：按名称归一化比较

**进度事件**：

| 阶段 | 进度 | 消息 |
|------|------|------|
| extracting | 0 | 开始提取入排标准... |
| extracting | 0-100 | 正在处理第 X/Y 块... |
| complete | 100 | 入排标准提取完成 |

### 2.7 extractSubjectData

从已上传的受试者文件中 AI 提取受试者数据。

| 项目 | 值 |
|------|-----|
| 通道 | `extract-subject-data` |
| 方向 | 渲染进程 → 主进程 |
| 请求参数 | 无（使用内存中的 `subjectFileInfo`） |
| 响应类型 | `SubjectData` |

**前置条件**：必须先调用 `uploadSubjectFile` 成功。

**响应结构 `SubjectData`**：

```typescript
{
  demographics: {
    subjectId: string;       // 受试者编号
    initials: string;        // 姓名缩写
    age: number;             // 年龄
    gender: string;          // 性别
    ethnicity?: string;      // 民族
  };
  vitalSigns: {
    type: string;            // 生命体征类型
    value: string;           // 值
    unit: string;            // 单位
    date?: string;           // 测量日期
  }[];
  medicalHistory: {
    condition: string;       // 疾病/症状
    onsetDate?: string;      // 发病日期
    status?: string;         // 当前状态
    notes?: string;          // 备注
  }[];
  medications: {
    medicationName: string;  // 药品名称
    dosage: string;          // 剂量
    frequency: string;       // 频率
    startDate?: string;      // 开始日期
    endDate?: string;        // 结束日期
    indication?: string;     // 适应症
  }[];
  rawText?: string;          // 原始文本
}
```

**合并策略**（多文件/多块结果）：
- 人口学信息：先到先得（补充缺失字段）
- 生命体征：按 `type+value` 去重
- 病史：按 `condition` 去重
- 用药记录：按 `medicationName` 去重

---

### 2.8 verifyEligibility

AI 核验受试者是否符合入排标准。

| 项目 | 值 |
|------|-----|
| 通道 | `verify-eligibility` |
| 方向 | 渲染进程 → 主进程 |
| 请求参数 | `criteria: Criterion[]`, `subjectData: SubjectData` |
| 响应类型 | `EligibilityReport` |

**响应结构 `EligibilityReport`**：

```typescript
{
  subjectId: string;                 // 受试者编号
  results: {
    criterionId: string;             // 标准ID
    criterionContent: string;        // 标准内容
    category: 'inclusion' | 'exclusion';  // 标准 category
    status: 'pass' | 'fail' | 'unknown';  // 核验结果
    evidence: string;                // 支持判定的证据
    confidence: number;              // 置信度 (0-1)
    notes?: string;                  // 备注
  }[];
  overallEligible: boolean;          // 总体是否符合
  summary: string;                   // 核验摘要
  verifiedAt: string;                // 核验时间 (ISO 8601)
}
```

**核验规则**：
- 入选标准：`status === 'pass'` 表示符合该条标准
- 排除标准：`status === 'fail'` 表示触发了该排除条件（应被排除）
- `overallEligible`：所有入选标准 pass 且所有排除标准不 fail 时为 `true`

### 2.9 onProcessingProgress

监听处理进度更新事件。

| 项目 | 值 |
|------|-----|
| 通道 | `processing-progress` |
| 方向 | 主进程 → 渲染进程（Push Event） |
| 回调参数 | `ProcessingProgress` |

**事件结构 `ProcessingProgress`**：

```typescript
{
  stage: 'uploading' | 'parsing' | 'extracting' | 'verifying' | 'complete' | 'error';
  progress: number;   // 0 - 100
  message: string;     // 进度描述文本
}
```

**注册方式**（渲染进程）：

```typescript
const unsubscribe = window.electronAPI.onProcessingProgress(
  (progress: ProcessingProgress) => {
    console.log(progress.stage, progress.progress, progress.message);
  }
);

// 取消监听
unsubscribe();
```

## 3. Preload 桥接接口

预加载脚本通过 `contextBridge.exposeInMainWorld` 暴露以下 API：

```typescript
interface ElectronAPI {
  // 配置管理
  getSettings(): Promise<AppConfig>;
  saveSettings(settings: Partial<AppConfig>): Promise<AppConfig>;

  // 连接测试
  testConnection(): Promise<ConnectionTestResult>;

  // 文件上传
  uploadProtocolFile(filePath: string): Promise<FileUploadResult>;
  uploadSubjectFile(filePath: string): Promise<FileUploadResult>;

  // AI 提取
  extractCriteria(): Promise<ProtocolData>;
  extractSubjectData(): Promise<SubjectData>;

  // 资格核验
  verifyEligibility(): Promise<EligibilityReport>;

  // 事件监听
  onProcessingProgress(callback: (progress: ProcessingProgress) => void): () => void;
}
```

**使用方式**（渲染进程）：

```typescript
// 读取配置
const settings = await window.electronAPI.getSettings();

// 上传文件
const result = await window.electronAPI.uploadProtocolFile('/path/to/file.pdf');

// 提取数据
const protocolData = await window.electronAPI.extractCriteria();

// 监听进度
const unsub = window.electronAPI.onProcessingProgress((p) => {
  updateProgress(p.progress, p.message);
});

// 清理
unsub();
```

## 4. 内部模块接口

### 4.1 Gateway 接口

#### invokeGateway

所有 Agent 调用 AI 模型的统一入口。

```typescript
function invokeGateway(request: GatewayRequest): Promise<GatewayResponse>;
```

**`GatewayRequest`**：

```typescript
{
  prompt: string;                // 用户提示词
  systemPrompt?: string;         // 系统提示词
  contentType: 'text' | 'image'; // 内容类型
  imageBase64?: string;          // 单图 base64（向后兼容）
  imageMimeType?: string;        // 单图 MIME 类型
  images?: Array<{               // 多图批量（优先于 imageBase64）
    base64: string;
    mimeType: string;
  }>;
  temperature?: number;          // 温度（默认 0.1）
  maxTokens?: number;            // 最大 token（默认 4096）
}
```

**`GatewayResponse`**：

```typescript
{
  content: string;              // AI 响应内容
  model: string;                // 实际使用的模型名称
  usage: {
    promptTokens: number;       // 输入 token 数
    completionTokens: number;   // 输出 token 数
    totalTokens: number;        // 总 token 数
  };
}
```

#### resetRouter

重置网关路由器单例，使配置变更生效。

```typescript
function resetRouter(): void;
```

#### getChunkTokens

获取当前文本模型的动态分块大小（tokens）。

```typescript
function getChunkTokens(): number;
```

**计算规则**：`Math.min(contextWindow × 0.6, 6000)`，基于 `MODEL_SPECS` 中文本模型的上下文窗口。

#### getMaxImagesPerCall

获取当前视觉模型的单次最大图片数。

```typescript
function getMaxImagesPerCall(): number;
```

**返回值**：`MODEL_SPECS` 中视觉模型的 `maxImagesPerCall` 字段。

### 4.4 文件工具接口

#### validateFilePath

验证文件路径安全性。

```typescript
function validateFilePath(filePath: string): Promise<FileValidationResult>;

// FileValidationResult
{
  valid: boolean;
  error?: string;        // 验证失败原因
```

```typescript
  pageCount?: number;    // PDF 页数
```

## 5. 外部 API 调用规范

### 5.1 智谱 AI API

**端点**：`POST https://open.bigmodel.cn/api/paas/v4/chat/completions`

**请求头**：

```
Content-Type: application/json
Authorization: Bearer <api_key>
```

**请求体**：

```json
{
  "model": "glm-5.1",
  "messages": [
    { "role": "system", "content": "..." },
    { "role": "user", "content": "..." }
  ],
  "temperature": 0.1,
  "max_tokens": 4096
}
```

**图片请求体**（单图）：

```json
{
  "model": "glm-4.6v-flash",
  "messages": [
    { "role": "user", "content": [
      { "type": "image_url", "image_url": { "url": "data:image/png;base64,..." } },
      { "type": "text", "text": "..." }
    ]}
  ],
  "temperature": 0.1,
  "max_tokens": 4096
}
```

**多图批量请求体**（如 glm-4v-plus，最多 5 张）：

```json
{
  "model": "glm-4v-plus",
  "messages": [
    { "role": "user", "content": [
      { "type": "image_url", "image_url": { "url": "data:image/png;base64,..." } },
      { "type": "image_url", "image_url": { "url": "data:image/png;base64,..." } },
      { "type": "image_url", "image_url": { "url": "data:image/png;base64,..." } },
      { "type": "text", "text": "..." }
    ]}
  ],
  "temperature": 0.1,
  "max_tokens": 4096
}
```

**限流与重试**：

Gateway 自动处理以下可重试错误，按指数退避重试（最多 5 次）：
- HTTP 429（限流）
- timeout（请求超时）
- ECONNABORTED（连接中断）
- ECONNRESET（连接重置）

退避序列（基础延迟 10 秒）：
- 第 1 次重试：等待 10 秒
- 第 2 次重试：等待 20 秒
- 第 3 次重试：等待 40 秒
- 第 4 次重试：等待 80 秒
- 第 5 次重试：等待 160 秒
- 超过 5 次后抛出 `AI API call failed after 5 retries: ...`

连续 API 调用之间至少间隔 1.5 秒，60 秒内最多 5 次请求。

**响应体**：

```json
{
  "choices": [{
    "message": { "content": "...", "role": "assistant" },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 100,
    "completion_tokens": 200,
    "total_tokens": 300
  },
  "model": "glm-5.1"
}
```

**超时配置**：
- 文本请求：120 秒
- 图片请求：180 秒

### 5.2 OpenAI 兼容 API

**端点**：`POST <用户配置的端点>`

请求/响应格式与智谱 API 相同（OpenAI Chat Completions 兼容格式）。

## 7. 错误码与错误处理

### 7.1 文件验证错误

| 错误消息 | 原因 |
|----------|------|
| 路径不能包含 .. | 检测到路径遍历尝试 |
| 文件不存在 | 文件路径无效 |
| 不支持的文件类型: {mimeType} | MIME 类型不在白名单中 |
| 文件大小超过限制 (最大 100MB) | 文件过大 |
| 文件为空 | 文件内容为空 |

### 7.2 AI 调用错误

| 错误消息 | 原因 |
|----------|------|
| API Key 未配置 | 未设置 API 密钥 |
| AI API call failed: {detail} | API 调用失败（网络、认证、额度等） |
| AI API call failed after 5 retries: {detail} | 连续 5 次重试后仍失败（限流、超时或网络错误） |
| 解析 AI 响应失败 | AI 返回的 JSON 格式无效 |

### 7.3 流程错误

| 错误消息 | 原因 |
|----------|------|
| 请先上传方案文件 | 未上传文件就调用提取 |
| 请先上传受试者文件 | 未上传文件就调用提取 |

## 8. AI 提示词模板

### 8.1 入排标准提取

**系统提示词**：`CRITERIA_SYSTEM_PROMPT`

角色：临床试验数据提取专家。要求完整提取入选/排除标准，保留编号，区分类别，返回 JSON。

**用户提示词（文本）**：`CRITERIA_EXTRACTION_PROMPT`

```
从以下临床试验方案文本中提取所有入选标准和排除标准。

方案文本：
{text}

请以JSON格式返回...
```

**用户提示词（图片）**：`CRITERIA_EXTRACTION_FROM_IMAGE_PROMPT`

无 `{text}` 占位符，直接要求从图片中读取并提取。

### 8.2 访视计划提取

**系统提示词**：`VISIT_SCHEDULE_SYSTEM_PROMPT`

**用户提示词**：`VISIT_SCHEDULE_PROMPT`

包含 `{text}` 占位符。请求 JSON 格式 `{ visitSchedules: [...] }`。

### 8.3 受试者数据提取

**系统提示词**：`SUBJECT_SYSTEM_PROMPT`

要求提取人口学信息、生命体征、病史、用药记录，数据必须与原文一致。

**用户提示词（文本）**：`SUBJECT_EXTRACTION_PROMPT`
**用户提示词（图片）**：`SUBJECT_EXTRACTION_FROM_IMAGE_PROMPT`

### 8.4 资格核验

**系统提示词**：`ELIGIBILITY_SYSTEM_PROMPT`

角色：受试者资格核验专家。逐条检查标准，给出 pass/fail/unknown，引用证据，给出置信度。

**用户提示词**：`ELIGIBILITY_VERIFICATION_PROMPT`

```
入选标准：
{inclusionCriteria}

排除标准：
{exclusionCriteria}

受试者数据：
{subjectData}

请逐条核验...
```
